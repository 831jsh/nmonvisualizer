Overview

NMON Visualizer can process IO Stat files from both AIX and Linux. But the correct command line switches must be used.

Linux

  • Must use the -t option to include timestamps.
    By default, timestamps will be in the default locale of the OS. To ensure consistency, the ISO 8601 format should be used. This can be changed in newer versions of iostat by exporting S_TIME_FORMAT=ISO. This has the added benefit of including a time zone in each timestamp. Note that this option is not avaiable on older versions of IO Stat. For these versions, the GUI will prompt for a time zone.
  • May use
    • -c, CPU data
    • -n, NFS data
    • -N, device mapper and LVM data
    • -x, extended disk statistics
    • -p, partition data
    • -k or -m to display KB or MB instead of blocks
      Note that this should be the same on all hosts so data between hosts can be compared.
  • Cannot use
    • -z which skips output of zeroed data
    • -h which outputs human readable NFS output

AIX

  • Must use the -T option to include timestamps.
    Note that AIX does not have an option to include the time zone like Linux. In addition, the time format does not include the date. So, AIX files will always prompt for this information.
  • May use -s to output a system information, including the hostname.
  • Cannot use
    • -D which outputs extended disk stats when -l is used
    • -m which outputs path info
    • -X which outputs in XML format
